7. Jelaskan Perbedaan format file PDF, HTML, XML dan XHTML


Jawaban:

PDF merupakan salah satu format dokumen digital yang banyak digunakan saat ini. Banyak orang lebih memilih menggunakan format ini karena lebih rapi dan tidak berubah saat dibuka menggunakan perangkat lainnya.

HTML atau Hypertext Markup Language adalah bahasa markup yang digunakan untuk membuat struktur halaman website agar dapat ditampilkan pada web browser.

XML atau Extensible Markup Language merupakan markup language yang diciptakan oleh World Wide Web Consortium. Bahasa ini digunakan untuk menyederhanakan pertukaran dan penyimpanan data.

XHTML adalah sebuah kombinasi dari HTML dan XML (Extensible Markup Language) dan merupakan bentuk sempurna (dalam arti, lebih ketat dan bersih dibandingkan HTML). Tujuan XHTML adalah menggantikan HTML.

11. Apa saja kegunaan dari Microsoft Word dan apa keunggulan format file Microsoft Office berbasis XML?​


kegunaan dari Microsoft Word:

1. Menulis dan mengedit dokumen: Microsoft Word digunakan untuk membuat, mengedit, dan memformat dokumen teks seperti surat, laporan, esai, resume, dan banyak lagi.

2. Formatting dan tata letak: Word menyediakan fitur-formatting yang kuat untuk mengubah tampilan dan tata letak teks, seperti pemformatan huruf, paragraf, spasi, penomoran, pengaturan margin, dan lainnya.

3. Gambar dan grafik: Word memungkinkan pengguna untuk menyisipkan gambar, grafik, dan elemen visual lainnya ke dalam dokumen. Hal ini berguna untuk membuat dokumen yang lebih interaktif dan informatif.

4. Alat kolaborasi: Word menyediakan fitur kolaborasi yang memungkinkan beberapa pengguna untuk bekerja pada dokumen yang sama secara bersamaan. Hal ini memudahkan kolaborasi tim dan revisi dokumen secara efisien.

5. Pembuatan tabel: Word memiliki fitur untuk membuat tabel yang memungkinkan pengguna untuk mengatur dan menyusun data dalam format tabel.

6. Pemformatan halaman: Word memungkinkan pengguna untuk mengatur halaman dokumen, seperti header dan footer, penomoran halaman, pengaturan ukuran kertas, dan lainnya.

Keunggulan format file Microsoft Office berbasis XML, seperti .docx untuk Word, antara lain:

1. Kompatibilitas: Format XML Microsoft Office kompatibel dengan berbagai versi Microsoft Word, memungkinkan dokumen dapat dibuka dan diedit dengan lancar di berbagai platform dan versi perangkat lunak.

2. Ukuran file yang lebih kecil: Format XML memungkinkan kompresi data yang lebih baik, sehingga ukuran file dokumen lebih kecil dibandingkan dengan format file sebelumnya seperti .doc. Hal ini membantu menghemat ruang penyimpanan dan mempercepat waktu pengiriman dokumen.

3. Integrasi dengan aplikasi lain: Format XML Microsoft Office memudahkan integrasi dengan aplikasi lain dalam ekosistem Microsoft Office. Data dan konten dari dokumen dapat dengan mudah diekstrak dan digunakan dalam aplikasi seperti Excel dan PowerPoint.

4. Ketersediaan fitur canggih: Format XML memungkinkan penambahan fitur-fitur baru dan lebih canggih pada aplikasi Microsoft Office. Dengan format ini, pengguna dapat memanfaatkan fitur-fitur seperti pemodelan data, pemrograman makro, penyesuaian tata letak, dan banyak lagi.

5. Pemulihan dokumen yang lebih baik: Format XML mendukung pemulihan dokumen yang lebih baik dalam kasus terjadinya kegagalan atau kerusakan file. Format ini dirancang dengan metadata yang lebih lengkap, memungkinkan pemulihan yang lebih baik dari kerusakan atau kesalahan file.

Dalam keseluruhan, format file Microsoft Office berbasis XML memberikan keunggulan dalam kompatibilitas, ukuran file yang lebih kecil, integrasi dengan aplikasi lain, ketersediaan fitur canggih, dan pemulihan dokumen yang lebih baik.

17. Apakah package Name pada android studio dapat dirubah atau tidak? jelaskan alasan nya​


Jawaban:

Package name pada Android Studio tidak dapat diubah setelah proyek telah dibuat. Package name adalah identitas unik yang digunakan untuk mengidentifikasi aplikasi di sistem Android dan di Google Play Store.

Alasan utama mengapa package name tidak dapat diubah adalah untuk menjaga konsistensi dan stabilitas aplikasi. Package name digunakan oleh sistem Android untuk menghubungkan aplikasi dengan konfigurasi, database, sumber daya, dan komponen lainnya. Jika package name diubah, akan ada banyak perubahan yang diperlukan di dalam kode aplikasi, file manifest, dan berbagai bagian lain dari proyek. Hal ini dapat menyebabkan masalah kompatibilitas, kesalahan kompilasi, atau bahkan kerusakan aplikasi.

Selain itu, package name juga digunakan oleh Google Play Store untuk mengidentifikasi aplikasi yang diunggah. Jika package name diubah, maka aplikasi tersebut akan dianggap sebagai aplikasi yang berbeda oleh Google Play Store. Hal ini dapat menyebabkan masalah dalam pembaruan dan penyebaran aplikasi, serta dapat membingungkan pengguna yang telah menginstal versi sebelumnya.

Dalam kasus tertentu di mana package name perlu diubah, solusi yang disarankan adalah membuat proyek baru dengan package name yang diinginkan dan menyalin kode, sumber daya, dan komponen lainnya ke proyek baru tersebut. Namun, ini memerlukan upaya yang signifikan dan perhatian terhadap perubahan yang diperlukan di berbagai bagian aplikasi.

Jadi, disarankan untuk memilih package name yang tepat dan sesuai dengan kebutuhan aplikasi pada awal pembuatan proyek di Android Studio.

37. Jelaskan Pemrograman Fungsional di Android Studio


Jawaban:

Ketika banyak orang berbicara tentang topik pemrograman fungsional, mereka selalu menyebutkan kata kunci 'fungsional' yang sering membingungkan. Mereka selalu menyebutkan fitur yang disuguhkan pemrograman fungsional seperti data yang tidak berubah, fungsi first-class dan optimisasi tail call. Mereka juga selalu menyebutkan teknik yang selalu digunakan dalam pemrograman fungsional seperti mapping, reducing, pipelining, recursing, currying dan fungsi level atas lainnya. Mereka juga selalu bilang keuntungan pemrograman fungsional seperti parallelization, lazy evaluation, dan determinism.

Namun jangan terpaku pada istilah tersebut, pemrograman fungsional mempunyai karakterisitik yang penting yaitu 'The absence of side effects'. Artinya bahwa pemrograman fungsional tidak bergantung pada data yang diluar fungsi, dan tidak merubah data yang ada di luar fungsi.

Sebagai contoh perbedaan fungsi yang termasuk ke dalam pemrograman fungsional dan bukan diimplementasikan dalam bahasa pemrograman Python sebagai berikut:

Tidak fungsional

a = 0

def increment():

global a

a += 1

Fungsional

def increment(a):

return a + 1

Pada contoh lain, pemrograman fungsional tidak melakukan iterasi dalam struktur data seperti list, stack dan lain-lain. Untuk melakukan iterasi terhadap data, teknik yang digunakan biasanya dengan fungsi map() dan reduce().

Cara kerjanya, fungsi map() berguna mengumpulkan item sebagai elemen baru dan disimpan dalam koleksi kosong. Fungsi map() melakukan iterasi untuk setiap koleksi dan menyimpan setiap nilai dalam koleksi baru yang kemudian dikembalikan sebagai koleksi baru.

Baca juga: Virtual Reality dan Perkembangannya

Sebagai contoh, dibawah ini merupakan implementasi kuadrat.

squares = map(lambda x: x * x, [0, 1, 2, 3, 4])

print squares

# result => [0, 1, 4, 9, 16]

Apa saja sih keuntungan yang disediakan pemrograman fungsional? Ini dia.

Pemrograman fungsional membolehkan kita untuk menulis kode dengan potensi bug yang rendah karena setiap komponen telah diisolasi.

Penggunaan rekursif dan fungsi first-class membolehkan kita untuk menampilkan bukti sederhana dari struktur kode.

Parallelization, lazy evaluation, determinism dan masih banyak lagi.

Pemrograman fungsional termasuk kedalam paradigma menulis kode tanpa merubah state. Alasan utamanya, pemrograman fungsional melakukan pemanggilan fungsi yang dapat menghasilkan hasil yang sama secara berturut-turut. Untuk melakukan implementasi pemrograman fungsional, kita bisa menggnakan berbagai bahasa pemrograman yang mendukung fungsi first-class.

Dengan demikian pemrograman fungsional adalah pemrograman yang mempunyai karakteristik 'The absence of side effects'. Artinya fungsi yang ditulis tidak bergantung pada data. Pemrograman fungsional juga membawa gaya baru dalam menulis kode sehingga bisa lebih menghemat baris kode. (yp)

Kelebihan Android Studio:

1. Lingkungan Pengembangan yang Komprehensif: Android Studio menyediakan lingkungan pengembangan yang komprehensif dan kaya fitur, termasuk editor kode yang kuat, analisis kode, debugger, dan emulator Android Studio yang terintegrasi. Hal ini membuat pengembangan aplikasi Android menjadi lebih mudah dan efisien.

2. Dukungan untuk Bahasa Pemrograman Kotlin: Android Studio secara resmi mendukung bahasa pemrograman Kotlin, yang merupakan alternatif modern dan ekspresif untuk Java dalam pengembangan aplikasi Android. Dukungan yang baik untuk Kotlin memungkinkan pengembang menghasilkan kode yang lebih bersih, efisien, dan aman.

3. Integrasi dengan SDK dan Alat Android: Android Studio terintegrasi dengan Android SDK (Software Development Kit) dan berbagai alat pengembangan Android lainnya, seperti Gradle build system, ProGuard untuk obfuscation, dan lain-lain. Integrasi ini mempermudah pengaturan proyek, manajemen dependensi, dan distribusi aplikasi.

4. Desain Antarmuka yang Menarik: Android Studio menyediakan editor layout visual yang memungkinkan pengembang untuk merancang antarmuka pengguna dengan mudah menggunakan sistem tata letak yang responsif dan dukungan untuk drag-and-drop. Pengembang juga dapat memvisualisasikan tampilan aplikasi mereka dalam berbagai perangkat Android yang berbeda.

5. Plugin dan Ekstensibilitas: Android Studio mendukung plugin yang memperluas fungsionalitasnya. Ada banyak plugin pihak ketiga yang tersedia untuk Android Studio yang dapat membantu pengembang meningkatkan produktivitas dan mengintegrasikan alat dan layanan lain.

Kekurangan Android Studio:

1. Membutuhkan Sumber Daya Komputer yang Tidak Sedikit: Android Studio adalah IDE yang membutuhkan sumber daya komputer yang cukup tinggi. Untuk menjalankannya secara lancar, diperlukan komputer dengan spesifikasi yang memadai, termasuk RAM yang cukup besar. Komputer dengan spesifikasi rendah mungkin mengalami kinerja yang lambat saat menjalankan Android Studio.

2. Memiliki Pembelajaran Awal yang Curam: Bagi pengembang yang baru memulai, Android Studio dapat memiliki kurva pembelajaran yang curam. Terdapat banyak konsep dan fitur yang harus dipelajari, terutama jika Anda baru dalam pengembangan aplikasi Android atau belum terbiasa dengan lingkungan pengembangan Java/Kotlin.

3. Emulator yang Lambat: Meskipun Android Studio menyediakan emulator Android Studio yang terintegrasi untuk menguji aplikasi, emulator ini dapat lambat dalam kinerjanya, terutama pada komputer dengan spesifikasi yang rendah. Dalam beberapa kasus, lebih baik menggunakan perangkat fisik untuk pengujian yang lebih responsif.

4. Memakan Ruang Disk yang Besar: Instalasi Android Studio dan proyek-proyek yang dibuat dengannya dapat memakan ruang disk yang besar. Ini dapat menjadi kendala jika Anda memiliki batasan penyimpanan pada komputer Anda.
